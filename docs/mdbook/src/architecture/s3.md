# S3 Blob Storage

orb-chrysa stores all blob data (container image layers, configs, uploaded artifacts)
in S3-compatible object storage. RustFS is used for local development.

## BlobStore Trait

The `BlobStore` trait defines the blob I/O interface:

```rust
#[async_trait]
pub trait BlobStore {
    async fn stat_blob(&self, digest: &Digest) -> Result<Option<BlobMeta>>;
    async fn get_blob(&self, digest: &Digest) -> Result<BlobStream>;
    async fn upload_blob(&self, digest: &Digest, data: Bytes) -> Result<()>;
    async fn delete_blob(&self, digest: &Digest) -> Result<()>;
}
```

## S3 Implementation

The `store/s3.rs` module implements `BlobStore` using the AWS SDK for Rust. It
supports any S3-compatible storage (AWS S3, MinIO, RustFS, Ceph RGW).

## Key Layout

```
blobs/sha256/<first-2-chars>/<full-digest>     # content-addressable blobs
uploads/<session-uuid>/data                     # in-progress uploads
raft-snapshots/<node-id>/latest.bin             # Raft state machine backups
```

## Download Modes

Two modes for serving blob downloads, configured via `storage.s3.redirect.enabled`:

### Proxy Mode (default)

orb-chrysa streams blob data from S3 to the client through the server. Memory usage
is `O(buffer_size)`. Suitable when S3 is not publicly accessible.

### Redirect Mode

orb-chrysa returns a 302 redirect to an S3 pre-signed URL. The client downloads
directly from S3. Lower server bandwidth usage but requires S3 to be reachable
from clients.

```toml
[storage.s3.redirect]
enabled = true
public_endpoint = "http://rustfs:9000"
expires_secs = 900
```
